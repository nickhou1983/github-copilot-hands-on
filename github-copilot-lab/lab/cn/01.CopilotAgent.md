## GitHub Copilot Lab

### 什么是 GitHub Copilot？

GitHub Copilot 是由 GitHub 和 OpenAI 共同开发的 AI 编程助手，它可以：

- **智能代码补全**：根据上下文自动生成代码
- **自然语言转代码**：通过描述需求生成代码实现
- **多语言支持**：支持 Python、JavaScript、TypeScript、Go 等多种编程语言
- **学习最佳实践**：基于数十亿行开源代码训练，提供高质量代码建议

### GitHub Copilot 的优势

1. **提升开发效率**：减少重复代码编写，专注业务逻辑
2. **降低学习曲线**：帮助快速掌握新框架和技术
3. **代码质量保证**：提供符合最佳实践的代码建议
4. **上下文感知**：理解项目结构和编码风格，生成一致的代码

### 什么是 Plan Mode（规划模式）？

GitHub Copilot 的 Plan Mode 是一种在实际编写代码之前，先让 Copilot 探索代码库、收集上下文、明确需求并制定可行实施方案的工作模式。它的核心价值是：

- **先思考再行动**：在动手写代码之前，先理解问题、分析方案
- **减少返工**：通过前期规划，避免实现过程中频繁推翻重来
- **方案透明**：让开发者清楚 Copilot 计划如何实现需求，在实施前进行确认
- **结构化输出**：生成包含具体步骤的实施计划，便于审核和跟踪

### Agent Mode vs Plan Mode

| 特性 | Plan Mode | Agent Mode |
| ---- | --------- | ---------- |
| 执行方式 | 先规划，确认后再实施 | 直接开始实施 |
| 适用场景 | 复杂功能、架构变更、PRD 规划 | 明确的小型任务 |
| 用户控制 | 实施前可审核和调整方案 | 实施过程中可中断 |
| 上下文收集 | 主动探索代码库 | 边做边收集上下文 |

### 在本 Lab 中的应用

在本实验中，我们将结合 Plan Mode 和 Agent Mode，体验完整的 **"规划 → 文档 → 实现"** 开发流程：

- 使用 Plan Mode 规划 PRD 结构和内容
- 使用 Agent Mode 生成完整的 PRD 文档
- 使用 Agent Mode 根据 PRD 实现项目代码

## Lab 步骤

### 第一步：使用 Plan Mode 规划 PRD

#### 1.1 目标

使用 GitHub Copilot 的 Plan Mode 规划产品需求文档（PRD）的结构和内容方案，在生成文档之前先确认方向

#### 1.2 操作步骤

1. **创建项目文件夹**

   ```bash
   mkdir travel-site
   cd travel-site
   ```

2. **切换到 Plan Mode**

   在 VS Code 中打开已创建好的文件夹 travel-site，打开 Copilot Chat 面板，点击模式选择下拉菜单，将模式切换为 **"Plan"**

   > 💡 **提示**：Plan Mode 下 Copilot 不会直接修改代码或创建文件，而是会先制定计划并等待你的确认

3. **描述产品需求，让 Copilot 规划 PRD**

   在 Plan 模式下输入以下提示词：

   ```text
   我要创建一个旅游网站，只包含一个页面用于浏览目的地。
   请帮我规划一份PRD文档的结构和关键内容。PRD需要包括网站介绍，页面的功能介绍，
   文件目录结构和技术栈（html和tailwind css）。
   注意：该网站使用响应式设计，元素清晰简单，页面中所需图片尽量从Unsplash和Pexels获取，
   设计风格用蓝色和绿色
   ```

4. **审核规划结果**

   仔细阅读 Copilot 生成的 PRD 规划方案，关注以下内容：

   - PRD 文档将包含哪些章节
   - 技术栈选择是否合理
   - 页面功能描述是否满足需求
   - 文件目录结构是否清晰

5. **提出调整意见（可选）**

   如果规划中有不满意的地方，可以继续对话进行调整，例如：

   ```text
   我希望PRD中增加页面交互说明部分，并且文件目录结构更加详细，请更新计划
   ```

6. **确认计划**

   当规划方案满意后，点击对话中的 **确认按钮**，让 Copilot 按计划执行

#### 1.3 验证

- Copilot 在 Plan Mode 下生成了完整的 PRD 规划方案
- 方案包含了文档结构、功能要点和技术选型
- 确认后 PRD 文件被创建

---

### 第二步：生成 PRD 文档

#### 2.1 目标

使用 Agent Mode 根据规划好的方案，生成完整的 PRD 文档

#### 2.2 操作步骤

1. **切换到 Agent Mode**

   将 Copilot Chat 模式从 "Plan" 切换回 **"Agent"**

2. **生成完整 PRD 文档**

   如果上一步 Plan Mode 已生成了 PRD 文件，检查内容是否完整。如果需要补充或重新生成，在 Agent 模式下输入以下提示词：

   ```text
   请根据之前的规划，创建一份完整的PRD文档并保存为 PRD.md 文件。
   确保包含网站介绍、功能详细描述、页面布局说明、文件目录结构和技术栈信息。
   ```

3. **保存 PRD**

   在 Copilot Chat 对话框，点击 **"keep"** 按钮，保存生成的 PRD 文档

#### 2.3 验证

- PRD.md 文件已创建
- 文档结构清晰，包含所有必要章节（网站介绍、功能描述、目录结构、技术栈等）
- 内容与 Plan Mode 中确认的规划方案一致

---

### 第三步：根据 PRD 实现项目代码

#### 3.1 目标

使用 Agent Mode 根据 PRD 文档实现旅游网站项目代码

#### 3.2 操作步骤

1. **添加 PRD 引用**

   在 GitHub Copilot 聊天窗口添加对已创建好的 PRD 文件的引用，方便后续生成代码时参考

2. **创建项目**

   在 VS Code 中打开 Copilot Chat，确保处于 **Agent 模式**，输入以下提示词：

   ```text
   请根据PRD的内容创建网站应用
   ```

3. **启动 Debug 日志**

   在 VS Code Copilot Chat 右上角，点击"..."，选择"Show Chat Debug View"
   ![Copilot 引用示例占位](../images/011.png)

4. **打开应用**

   在 VS Code 中打开 Copilot Chat，选择 Agent 模式，输入以下提示词：

   ```text
   在浏览器中打开主页
   ```

5. **处理错误**

   生成的项目遇到任何问题，请继续使用 Copilot Chat 进行修复，直到项目可以成功运行。

#### 3.3 验证

- 网站应用被成功创建
- 功能符合 PRD 定义
- 观察 GitHub Copilot Chat 的 Debug 日志，了解上下文是如何传递的
