## GitHub Copilot Lab

### What is Plan Mode?

GitHub Copilot's Plan Mode is a working mode that allows Copilot to explore the codebase, gather context, clarify requirements, and create an actionable implementation plan before writing any code. Its core values are:

- **Think Before Acting**: Understand the problem and analyze solutions before writing code
- **Reduce Rework**: Avoid frequent do-overs during implementation through upfront planning
- **Transparent Approach**: Let developers see how Copilot plans to implement requirements and confirm before execution
- **Structured Output**: Generate implementation plans with concrete steps for easy review and tracking

### When to Use Plan Mode

1. **Adding New Functionality**: Where should the code go? What patterns to follow?
2. **Multiple Valid Approaches**: Choosing between technologies, patterns, or strategies
3. **Modifying Existing Behavior**: Unclear what should change or what side effects exist
4. **Architectural Decisions**: Choosing between design patterns or integration approaches
5. **Cross-File Changes**: Refactoring, migrations, or cross-cutting concerns
6. **Underspecified Requirements**: Need to explore before understanding scope

### Application in This Lab

In this lab, we will use GitHub Copilot's Plan Mode to:
- Plan the implementation of a new feature in the travel website project
- Experience the "plan first, implement second" development workflow

## Lab Steps

### Step 1: Enable Plan Mode

#### 1.1 Objective
Learn how to switch to Plan Mode in GitHub Copilot Chat

#### 1.2 Operation Steps

1. **Open the Travel Website Project**

   Open the travel website project created in Lab 01 in VS Code

2. **Switch to Plan Mode**

   In the Copilot Chat panel, click the mode selection dropdown and switch from "Agent" to "Plan"

   > 💡 **Tip**: In Plan Mode, Copilot will not directly modify code. Instead, it will create a plan and wait for your confirmation

---

### Step 2: Plan a New Feature Using Plan Mode

#### 2.1 Objective
Use Plan Mode to plan how to add "destination search and filter" functionality to the travel website

#### 2.2 Operation Steps

1. **Describe the New Requirement**

   In Copilot Chat (ensure Plan mode is selected), enter the following prompt:
   ```
   I want to add destination search and filter functionality to the travel website. 
   Users should be able to search destinations by keywords and filter by region 
   or price range. Please analyze the current project structure and create an implementation plan.
   ```

2. **Review the Plan**

   Copilot will:
   - Automatically explore relevant files in the project
   - Analyze the current code structure
   - Generate an implementation plan with concrete steps

3. **Review the Implementation Plan**

   Carefully read the plan generated by Copilot, paying attention to:
   - Which files will be modified
   - What new components or functions will be added
   - Whether the implementation steps are reasonable

4. **Provide Adjustment Feedback**

   If there are aspects of the plan you'd like to change, continue the conversation:
   ```
   I'd like the search feature to use debounce handling, and the filter criteria 
   should be in a sidebar format. Please update the plan.
   ```

5. **Confirm and Execute the Plan**

   Once satisfied with the plan, click the confirm button in the conversation. Copilot will start implementing the code changes according to the plan

#### 2.3 Verification
- Copilot generated a complete implementation plan
- The plan includes specific file modification lists and step descriptions
- Code changes after confirmation match the plan content

---

### Step 3: Compare Agent Mode and Plan Mode

#### 3.1 Objective
Understand the differences between Plan Mode and Agent Mode and their respective use cases

#### 3.2 Operation Steps

1. **Switch Back to Agent Mode**

   Switch the Copilot Chat mode back to "Agent"

2. **Execute a Similar Task Using Agent Mode**

   In Agent mode, enter the following prompt:
   ```
   Add a "favorite destinations" feature to the travel website where users can 
   click a favorite button to add destinations to their favorites list
   ```

3. **Observe the Differences**

   Notice the differences between Agent Mode and Plan Mode:

   | Feature | Plan Mode | Agent Mode |
   |---------|-----------|------------|
   | Execution | Plan first, implement after confirmation | Start implementing directly |
   | Best For | Complex features, architectural changes | Clear, small tasks |
   | User Control | Review and adjust before implementation | Can interrupt during implementation |
   | Context Gathering | Proactively explores codebase | Gathers context while working |

#### 3.3 Verification
- Understand the execution flow differences between the two modes
- Be able to choose the appropriate mode based on task complexity
